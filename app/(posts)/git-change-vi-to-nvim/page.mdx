# Git commit 편집기로 vi 대신 nvim 사용

Git에 소스 코드를 올리기 위해서는 `git commit` 명령어를 사용해야 합니다. 이 명령어는 변경 사항을 커밋하여 로컬 저장소에 기록하는 과정입니다. 커밋을 할 때는 간단한 메시지를 작성하여 해당 변경 사항에 대한 설명을 추가할 수 있습니다.

## 간단한 커밋 메시지

커밋을 수행할 때 메시지를 간단히 작성하려면 다음과 같이 입력합니다:

```bash
git commit -m "커밋 메시지"
```

여기서 `-m` 옵션은 메시지를 직접 입력할 수 있게 해주며, 이 경우 제목만 입력하는 형태가 됩니다. 예를 들어:

```bash
git commit -m "버그 수정"
```

이와 같이 입력하면 제목만 있는 커밋이 생성됩니다.

## 본문 작성하기

만약 커밋 메시지에 본문도 추가하고 싶다면, `-m` 옵션을 사용하지 않고 `git commit` 명령어만 입력하면 됩니다. 이 경우 Git은 기본 편집기(기본적으로 `vi`)를 열어주고, 여기에서 제목과 본문을 입력할 수 있습니다.

```bash
git commit
```

이 명령어를 입력하면 `vi` 편집기가 열리며, 다음과 같은 화면이 나타납니다:

![](https://i.imgur.com/oyPdzKw.png)

여기에서 제목과 본문을 입력할 수 있습니다.

## nvim으로 기본 에디터 변경하기

기본적으로 Git은 `vi` 를 편집기로 사용합니다. 하지만 `nvim`([Neovim](https://neovim.io/))을 선호하는 사용자라면, Git의 기본 편집기를 `vi` 에서 `nvim` 으로 변경할 수 있습니다. 이를 위해 아래의 명령어를 입력합니다:

```bash
git config --global core.editor "nvim"
```

위 명령어를 실행하면 Git의 기본 편집기가 `nvim` 으로 설정됩니다.

![](https://i.imgur.com/NIw1qAf.png)

## 설정 확인하기

설정이 제대로 되었는지 확인하려면 다음 명령어를 입력합니다:

```bash
git config --global --get core.editor
```

이 명령어를 실행하면 현재 설정된 Git의 편집기가 출력됩니다. 만약 `nvim` 이 제대로 설정되었다면, 아래와 같은 결과를 볼 수 있습니다:

```
nvim
```

이렇게 설정을 마치면, 이제부터 Git 커밋 시 `nvim` 편집기를 사용하여 편리하게 커밋 메시지를 작성할 수 있습니다.
